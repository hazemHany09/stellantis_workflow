# Lead → Subagent Contract

The JSON shape the lead agent embeds at the top of `.harness/SubAgent/<agent-name>.md` when spawning a subagent. The subagent reads this block before doing anything else.

## Shape

```json
{
  "contract_version": "1.0",
  "run_id": "<run-id>",
  "agent_name": "<agent-name>",
  "agent_working_file": "runs/<run-id>/.harness/SubAgent/<agent-name>.md",
  "kb_dataset_id": "<ragflow-dataset-id>",
  "car_identity": {
    "brand": "<string>",
    "model": "<string>",
    "model_year": "<integer>",
    "market_input": "<string, as submitted by user>",
    "market_canonical": "<ISO alpha-2 or zone code>",
    "resolved_trim": "<string — the full-option trim name>"
  },
  "category": "<category name from params.csv, verbatim>",
  "partition_index": "<integer, 1-based>",
  "parameter_ids": ["<stable id of each parameter in the subset>"],
  "parameters": [
    {
      "id": "<stable id; see below>",
      "name_internal": "<from params.csv Parameter column>",
      "name_optimised": "<from params.csv Optimised Parameter Name>",
      "description": "<from params.csv Parameter Description>",
      "category_description": "<from params.csv Category Description, if present>",
      "schema_type": "<one of SchemaType enum values>",
      "criteria": {
        "Basic":  "<text or null>",
        "Medium": "<text or null>",
        "High":   "<text or null>"
      }
    }
  ],
  "budgets": {
    "main_loop_iterations_max": 3,
    "fallback_loop_iterations_max": 3
  },
  "rules_refs": {
    "enums": "src/references/enums.md"
  },
  "output_schemas": {
    "per_parameter": "src/templates/intermediate-parameter-record.json.tmpl",
    "result_envelope": "src/references/subagent-to-lead-contract.md",
    "traceability_block": "src/templates/traceability-block.json.tmpl"
  },
  "tool_allowlist": [
    "retrieval",
    "retrieve_knowledge_graph",
    "list_chunks",
    "download_attachment",
    "doc_infos",
    "get_metadata_summary",
    "set_doc_metadata",
    "batch_update_doc_metadata"
  ],
  "created_at": "<ISO-8601 UTC timestamp>"
}
```

## Field notes

* **`agent_name`** — matches the filename stem of the working file. Derived from `<category-slug>-<partition-index>`.
* **`parameter_ids`** — stable id formed by `<category-slug>:<row-index-in-params.csv>`. Used for cross-referencing.
* **`parameters`** — full self-contained parameter descriptors so the subagent never needs to read `params.csv` again. Criteria cells that are empty in the CSV are emitted as JSON `null`, which is how the subagent knows the tier is not in the applicable set.
* **`budgets`** — per [`skills/subagent-classification-loop/SKILL.md`](../skills/subagent-classification-loop/SKILL.md). Main loop = query→read→update cycles; fallback loop = targeted hybrid retrieval on still-unresolved parameters.
* **`tool_allowlist`** — the exact list enforced by the subagent classification skill. Any tool not on this list is a bug.

## Writing rule

The contract is embedded verbatim in a fenced `json` block at the top of the subagent working file. The first non-blank content of the file looks like:

````markdown
<!-- AUTOGENERATED BY LEAD — DO NOT EDIT -->
```json
{ ...contract json... }
```
<!-- END CONTRACT -->

# Subagent working file
... (subagent appends below this line)
````

The subagent **must not** edit the contract block; it writes only after the `END CONTRACT` marker.

## Subagent obligation: loaded_skills

When the subagent finishes (before emitting its result envelope), it **must** record which skills it loaded during the run in the result envelope's `loaded_skills` field. This field is required and enables the lead to consolidate skill usage for auditing and debugging. See `src/references/lead-consolidation-loaded-skills.md` for consolidation rules.
