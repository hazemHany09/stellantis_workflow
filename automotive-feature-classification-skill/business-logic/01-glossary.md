# 01 — Glossary

Precise definitions. Every other document in this set must use these terms exactly.

There is a known term collision. The word *domain* can mean two unrelated things depending on the speaker:

- When talking about **search / the web**, a *domain* is a website host (e.g. `bmw.com`, `insideevs.com`). In this document set that concept is always called a **source site**.
- When talking about **features of the car**, the client may call the functional area (EV, Infotainment, ADAS…) a *domain* or a *category*. The reference CSV column is literally `Domain / Category`. In this document set that concept is always called a **category**.

> ⚠️ **Client-facing note.** The client may continue to say *domain* when they mean a feature category. Any interface that talks to the client (prompts, messages, reports) must accept the word *domain* as a synonym of **category** when the context is features, and must never confuse it with **source site**. Internal documents, code, and tool arguments use **category** and **source site** only.

| Term                         | Definition                                                                                                                                                                                                                            |
| :--------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Car**                      | A (brand, model, model-year, trim) tuple. Within this skill, `trim` is always the full-option trim.                                                                                                                                   |
| **Full-option trim**         | The highest-equipment variant offered for a given model in a given market. Only full-option trims are analysed.                                                                                                                       |
| **Category**                 | A high-level functional area of the car, as defined in the reference parameter list (e.g. `EV`, `INFOT.`, `ADAS`). Corresponds to the `Domain / Category` column of `artifacts/params.csv`. The client may informally call this a *domain*; accept that as a synonym in client-facing contexts, but never use *domain* for this concept in internal documents, code, or tool arguments. |
| **Parameter**                | An individual feature or capability under evaluation (e.g. `Trip planner / EV Routing`, `Battery Preconditioning`). One row of the reference parameter list.                                                                          |
| **Level**                    | The implementation tier assigned to a supported parameter for a given car. One of: `Basic`, `Medium`, `High`.                                                                                                                         |
| **Applicable level set**     | The subset of {Basic, Medium, High} for which the reference parameter list actually defines a criterion. Some parameters define only `High`; some define `Basic` and `High`; some define all three. A level is assignable **only if it is in the applicable set**. See `02-parameters-and-levels.md`. |
| **Criterion**                | The text in a level cell (`Basic — Criterion`, `Medium — Criterion`, `High — Criterion`) that describes what a car must deliver to qualify for that level.                                                                            |
| **Source site**              | A website (identified by its host, e.g. `insideevs.com`) from which evidence may be collected. This is what is usually called a "domain" in web / search contexts. In this document set, use **source site** in every context — even when the client says "domain". |
| **Source site list**         | The set of source sites that will be used for a given analysis. Can be **client-supplied** (the client hands over a list) or **agent-discovered** (the agent produces a list when the client does not supply one).                   |
| **Source document**          | A concrete artefact (HTML page, PDF, article) retrieved from a source site and intended for ingestion into the knowledge base.                                                                                                        |
| **Evidence**                 | A passage or chunk extracted from a source document that supports a presence decision, a level decision, or both, for a specific parameter on a specific car.                                                                         |
| **Citation**                 | A reference attached to a decision, pointing back to the source document and — where the knowledge base supports it — the specific chunk that served as evidence.                                                                      |
| **Knowledge base**           | An external system that ingests source documents, parses them into retrievable chunks, and answers semantic-retrieval queries. Referred to in the abstract; tool-specific details live in `07-tools-and-constraints.md`.               |
| **Ingestion**                | The process by which the knowledge base parses a newly uploaded source document and makes it available for retrieval. Ingestion is asynchronous and must complete for every source document before the classification stage starts.  |
| **Source approval gate**     | A mandatory pause in which the agent presents the proposed source site list (and/or source document list) to the client and waits for explicit approval before downloading and ingesting anything.                                    |
| **Harness**                  | The downstream classification stage that consumes retrieved evidence and emits, per parameter: presence decision, level decision, presence justification, level justification, citations. Its internal logic is out of scope for this document set; see `06-harness-interface.md`. |
| **Deliverable**              | The final structured result produced for one car: the full parameter-by-parameter classification with justifications and citations.                                                                                                   |
